# 🛒 Quick Commerce Data Analysis using Python

## 📌 Project Overview

This project analyzes operational and customer data from various Quick Commerce platforms to uncover insights related to revenue generation, customer satisfaction, delivery performance, discount effectiveness, and city-wise business performance.

The project demonstrates a complete data analysis workflow including data cleaning, handling missing values, exploratory data analysis (EDA) and dashboard creation using Python.

---

## 🎯 Business Problem

Quick commerce companies operate in a highly competitive environment where customer satisfaction and operational efficiency are critical.

This project aims to answer key business questions such as:

* Which company generates the highest revenue?
* Which platform has the highest Average Order Value (AOV)?
* How do customer ratings vary across companies?
* Does delivery time impact delivery partner ratings?
* Which cities perform best for different companies?
* How do discounts influence customer purchases?
* Which companies operate most efficiently?

---

## 📂 Dataset Information

The dataset contains **1 million order records** from multiple quick commerce platforms.

### Features Included

| Column Name             | Description                         |
| ----------------------- | ----------------------------------- |
| Order_ID                | Unique order identifier             |
| Company                 | Quick commerce platform/company     |
| City                    | City where the order was placed     |
| Customer_Age            | Age of customer                     |
| Order_Value             | Total order amount                  |
| Delivery_Time_Min       | Delivery time in minutes            |
| Distance_Km             | Distance travelled for delivery     |
| Items_Count             | Number of items ordered             |
| Product_Category        | Category of product purchased       |
| Payment_Method          | Mode of payment used                |
| Customer_Rating         | Customer satisfaction rating        |
| Discount_Applied        | Discount percentage/value applied   |
| Delivery_Partner_Rating | Rating assigned to delivery partner |

---

## 🧹 Data Cleaning & Preprocessing

The following preprocessing steps were performed:

* Randomly sampled **50,000 records** from the original dataset for analysis.
* Checked data types and dataset structure.
* Identified missing values.
* Filled missing values in:

  * **City** using mode.
  * **Items_Count** using median.
  * **Customer_Rating** using company-wise average ratings.
  * **Delivery_Partner_Rating** using delivery-time-wise average ratings.
* Rounded numerical columns for consistency.
* Converted `Order_ID` into string format.
* Created custom delivery speed categories using delivery time.
* Exported cleaned dataset for further analysis.

## 📊 Exploratory Data Analysis

### 1. Company Revenue Analysis

* Calculated total revenue generated by each company.
* Identified highest and lowest revenue-generating companies.

### 2. Average Order Value (AOV) Analysis

* Compared average order value across all companies.

### 3. Customer Satisfaction Analysis

* Analyzed average customer ratings company-wise.
* Visualized rating distribution using boxplots and grouped bar charts.

### 4. Delivery Performance Analysis

* Studied the relationship between delivery time and delivery partner ratings.
* Categorized deliveries into multiple delivery speed groups.

### 5. City-wise Business Performance

Analyzed:

* Total Orders
* Average Customer Rating
* Average Delivery Time
* Total Revenue

to identify top-performing cities for each company.

### 6. Discount Effectiveness Analysis

Evaluated the impact of discounts on:

* Average Order Value
* Total Items Purchased

### 7. Operational Efficiency Analysis

Compared companies based on:

* Average Delivery Time
* Order Volume

to assess operational efficiency.
